    Returns a Relay Connection to a collection of Action objects.

Access is permitted when any of the following condition(s) are met: 1. Always allowed.

## Input fields

| Argument                | Type                                                       | Description                                                                                        |
| ----------------------- | ---------------------------------------------------------- | -------------------------------------------------------------------------------------------------- |
| `filterQuery`           | [`JSONString`](api/schema/scalar/JSONString.mdx)           | JSON structure defining a conditional logic expression tree to use for including specific Actions. |
| `before`                | [`String`](api/schema/scalar/String.mdx)                   | Specifies a cursor for use in combination with `last` to implement backward pagination.            |
| `after`                 | [`String`](api/schema/scalar/String.mdx)                   | Specifies a cursor for use in combination with `first` to implement forward pagination.            |
| `first`                 | [`Int`](api/schema/scalar/Int.mdx)                         | A non-negative integer that specifies to return at most `first` edges after the `after` cursor.    |
| `last`                  | [`Int`](api/schema/scalar/Int.mdx)                         | A non-negative integer that specifies to return at most `last` edges before the `before` cursor.   |
| `offset`                | [`Int`](api/schema/scalar/Int.mdx)                         |                                                                                                    |
| `key`                   | [`String`](api/schema/scalar/String.mdx)                   | Filter for objects where key matches the specified value.                                          |
| `key_Icontains`         | [`String`](api/schema/scalar/String.mdx)                   | Filter for objects where key contains the specified value (case insensitive).                      |
| `key_In`                | [`[String]`](api/schema/scalar/String.mdx)                 | Filter for objects where key is contained in the list of specified values.                         |
| `label_Icontains`       | [`String`](api/schema/scalar/String.mdx)                   | Filter for objects where label contains the specified value (case insensitive).                    |
| `label_Fulltext`        | [`String`](api/schema/scalar/String.mdx)                   | Filter for objects where label.fulltext matches the specified value.                               |
| `description_Icontains` | [`String`](api/schema/scalar/String.mdx)                   | Filter for objects where description contains the specified value (case insensitive).              |
| `component`             | [`ID`](api/schema/scalar/ID.mdx)                           | Filter for objects where component matches the specified ID.                                       |
| `isTrigger`             | [`Boolean`](api/schema/scalar/Boolean.mdx)                 | Filter for objects where isTrigger matches the specified value.                                    |
| `isCommonTrigger`       | [`Boolean`](api/schema/scalar/Boolean.mdx)                 | Filter for objects where isCommonTrigger matches the specified value.                              |
| `isDataSource`          | [`Boolean`](api/schema/scalar/Boolean.mdx)                 | Filter for objects where isDataSource matches the specified value.                                 |
| `dataSourceType`        | [`String`](api/schema/scalar/String.mdx)                   | Filter for objects where dataSourceType matches the specified value.                               |
| `orderBy`               | [`ActionOrder`](api/schema/input_object/ActionOrder.mdx)   | DEPRECATED. Prefer using sort_by instead as it allows ordering by many fields.                     |
| `sortBy`                | [`[ActionOrder]`](api/schema/input_object/ActionOrder.mdx) | Allows specifying many field and direction pairs to sort results by.                               |

## Return fields ([ActionConnection!](api/schema/object/ActionConnection.mdx))

###### <b>edges</b> ([[ActionEdge]!](api/schema/object/ActionEdge.mdx))

List of edges containing the nodes in this connection.

###### <b>nodes</b> ([[Action]!](api/schema/object/Action.mdx))

List of nodes in this connection.

###### <b>pageInfo</b> ([PageInfo!](api/schema/object/PageInfo.mdx))

Pagination data for this connection.

###### <b>totalCount</b> ([Int!](api/schema/scalar/Int.mdx))

Total count of nodes available.
