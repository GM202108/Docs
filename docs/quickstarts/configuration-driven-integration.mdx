En esta guía rápida veremos cómo puede utilizar varios tipos de variables de configuración y conexiones para controlar la lógica de su integración.
Esto ayuda a ilustrar cómo sus clientes pueden tener instancias de la misma integración desplegadas para ellos, con diferentes configuraciones.

## Variables de configuración básicas

El tipo más básico de variable de configuración es una simple cadena.
Una variable de configuración de cadena puede tener un valor por defecto opcional, y puede añadir texto de ayuda **Description** para indicar al usuario final qué tipo de cadena espera la variable de configuración.

Por ejemplo, podemos crear una variable de configuración que pregunte al usuario por su punto final de API:

## Creación de menús desplegables y ramificación de opciones

Si desea limitar las opciones entre las que puede elegir un usuario final, seleccione **picklist** como tipo de datos de su variable de configuración.
Puede generar una lista de opciones que el usuario puede elegir:

<Screenshot
  filename="quickstarts/configuration-driven-integration/picklist-config-var.png"
  alt="Variable de configuración Picklist"
  maxWidth="500px"
/>

Al desplegar una instancia de la integración, tu usuario verá un menú desplegable con las opciones que hayas creado:

<Screenshot
  filename="quickstarts/configuration-driven-integration/picklist-config-wizard.png"
  alt="Introducción de listas de selección en el asistente de configuración"

Estas variables de configuración se pueden utilizar como se utilizan las variables de configuración de cadena (arriba).

También puede controlar la lógica de bifurcación desde una lista de selección añadiendo una bifurcación para cada opción de la lista de selección.

## Preguntar por listas de valores

A veces es útil que el usuario introduzca uno o más valores para una variable de configuración (piensa en "arrays" en JavaScript o "listas" en Python).
Cree una variable de configuración y seleccione **Lista** como **Tipo de variable de configuración**.

<Screenshot
  filename="quickstarts/configuration-driven-integration/list-config-var.png"
  alt="Lista de cadenas en una variable de configuración"
  maxWidth="700px"
/>

Los usuarios pueden introducir uno o más valores para la variable de configuración

<Screenshot
  filename="quickstarts/configuration-driven-integration/list-config-wizard.png"
  alt="Listar var config en el asistente de configuración"
  maxWidth="550px"
/>

Si haces referencia a esa variable de configuración en los pasos de tu integración, el código que respalda tu paso recibirá una matriz de cadenas en lugar de una única cadena.

Es posible que desee pedir a un usuario con un "sí / no" pregunta, y la unidad de alguna lógica basada en su respuesta.
Primero, cree una entrada booleana.
