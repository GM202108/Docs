id: tap-into-power-flows
title: Aproveche el poder de los flujos
description: Entender cómo funcionan los flujos dentro de las integraciones puede abrir vías más eficientes para construirlas.
---

Una sola integración puede tener un [flujo único](/docs/building-integrations/#flows-in-integrations) (para las integraciones más simples). Sin embargo, muchas integraciones se benefician al tener [múltiples flujos](/docs/quickstarts/building-an-integration-with-multiple-flows/). Por ejemplo, una integración bidireccional tiene flujos tanto de importación como de exportación.

El uso de diferentes flujos dentro de una sola integración permite que cada flujo tenga sus propios puntos finales, desencadenantes y pasos. Incluso puedes combinar tipos dentro de una integración donde algunos flujos se ejecutan según horarios, algunos se basan en webhooks y otros se ejecutan automáticamente al implementarse/activarse.

Las instancias (una integración implementada para un solo cliente) pueden tener puntos finales comunes, o puedes configurar el sistema con un punto final separado para cada instancia. De manera similar, los flujos pueden tener un punto final común, o cada uno puede tener uno diferente. Es práctica común utilizar flujos de preprocesamiento para dirigir los datos al flujo correcto.

Usar múltiples flujos en una sola integración, en lugar de construir una integración separada para cada flujo, también significa que los clientes solo necesitan activar una sola integración dentro del [marketplace](/integration-marketplace/).
