---
id: retry-replay-save-time
title: Reintentar y repetir ahorra tiempo
description: Resolver los problemas de integración no es muy divertido, pero utilizar la función de reintento y repetición puede ahorrarle mucho tiempo.
---

No todos los sistemas son accesibles todo el tiempo. Por lo tanto, cuando tenga sentido, debes [crear una función de reintento](/docs/building-integrations/#integration-retry-configuration) en una integración. El reintento permite que el sistema reenvíe automáticamente la carga útil al sistema de destino cada pocos minutos o incluso según una progresión geométrica (reintento en 2 minutos, 4 minutos después, 8 minutos después, etc.). Establecer el reintento en la integración asegura que es probable que tu integración se ejecute con éxito incluso si el sistema de destino o algo en la integración sufre una interrupción temporal.

El uso de la función de reintento también asegura que no involucres a tus desarrolladores para resolver un problema de integración hasta que hayas determinado que el problema es más grave que una breve interrupción del sistema. Incluir la función de reintento en las integraciones evita que muchos problemas transitorios lleguen al soporte.

Sin embargo, si el error persiste (por ejemplo, parece que el sistema de destino está activo, pero la integración sigue arrojando algún tipo de mensaje de error interno), es posible que necesites pasar [de la función de reintento a la función de repetición](/docs/instances/#instance-execution-retry-and-replay).

A diferencia del reintento, la repetición no es algo que se pueda hacer automáticamente dentro de una integración. En cambio, es una función que invocas manualmente haciendo clic en el botón de repetición en la interfaz de usuario para esa instancia. Al hacer esto, ejecutas la integración completa con la misma carga útil que antes, lo que te permite depurar al ver todos los pasos en la integración mientras se ejecutan para determinar exactamente qué está funcionando y dónde se producen errores.
