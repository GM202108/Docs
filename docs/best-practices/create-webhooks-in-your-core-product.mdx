---
id: create-webhooks-in-your-core-product
title: Crear Webhooks en su producto principal
description: El valor de configurar su aplicación principal para utilizar webhooks para la mensajería saliente.
---

Los webhooks en tu aplicación principal te permiten simplificar las integraciones de exportación. Configurar webhooks puede agregar bastante tiempo de desarrollo a la primera integración de exportación, pero las integraciones subsiguientes se pueden completar mucho más rápido gracias a los webhooks. Y, hablando de rapidez, los webhooks ayudan a que tu exportación funcione casi en tiempo real.

Los webhooks tampoco están limitados en cuanto a los puntos finales, por lo que se pueden utilizar para exportar datos a múltiples destinos. Por ejemplo, si tienes un webhook que [se activa](/docs/integration-triggers/#webhook-triggers) cada vez que hay una actualización en tu sistema de inventario, y dispara un payload que incluye datos de inventario actualizados (incluso si algunos de esos datos son irrelevantes para la integración actual), puedes filtrar fácilmente esos datos dentro de la integración para lo que necesita el sistema de destino. Entonces, al crear la siguiente integración para actualizaciones de inventario, ya tienes el payload con todo incluido (que ahora puedes filtrar para un conjunto de datos diferente para tu segunda integración).

Una vez que tengas configurados los webhooks para tu aplicación principal, no deberías necesitar casi tanta ayuda de los desarrolladores para crear integraciones de exportación adicionales. De hecho, tus técnicos no desarrolladores deberían poder construir integraciones de exportación que sigan el mismo patrón que las integraciones existentes. Esto libera a tus desarrolladores para trabajar en el producto principal en lugar de en las integraciones.
