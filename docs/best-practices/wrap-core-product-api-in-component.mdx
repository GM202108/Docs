title: Envolver la API de productos básicos en un componente
description: Las integraciones con tu producto principal son mucho más sencillas si has hecho el trabajo previo de configurar la API y envolverla con un componente.
---

Realizar algo una vez y hacerlo bien suele ser la mejor aproximación. Con eso en mente, una de las cosas más importantes que puedes hacer es [construir un componente que envuelva tu API](/docs/quickstarts/wrapping-an-api-in-a-component/). Luego, podrás reutilizar este componente compartido para cada integración de importación.

Dada la cantidad no trivial de trabajo requerido para construir este componente, la primera integración llevará un tiempo. Cuando se trata de tu segunda y tercera integración de importación, podrás reutilizar en gran medida el componente que construiste. Dependiendo de lo que necesites importar de la aplicación de terceros, es posible que necesites hacer un poco de desarrollo adicional, pero es probable que sea mínimo en ese momento. Sin embargo, cuando construyas tu enésima integración que utiliza este componente, el tiempo ahorrado compensará con creces ese desarrollo inicial.

Una vez que tus desarrolladores hayan escrito este componente compartido y lo hayan utilizado para construir algunas integraciones, el componente debería estar listo para que tus técnicos no desarrolladores construyan muchas más integraciones de importación, sin molestar a tu equipo de desarrollo para obtener más ayuda o código.
